# 🔧 CHANGE 1 — ДОБАВИТЬ PRO MODE BLOCK

В начало отчёта (сразу после SUMMARY) добавить:

=== PRO MODE: CRITICAL FIXES ===

Rules:

* ALWAYS output EXACTLY 5 fixes
* Order by impact (most critical first)
* Include ONLY high-impact issues
* Each fix must be actionable and technical

Format:

1. [critical issue + action]
2. [critical issue + action]
3. [critical issue + action]
4. [critical issue + action]
5. [critical issue + action]

Example:

1. Increase loudness to -8 to -9 LUFS using a limiter
2. Fix kick/bass masking using sidechain compression
3. Tighten low-end with high-pass at 80 Hz on bass
4. Reduce harshness around 3–5 kHz with EQ cut
5. Improve drop impact with arrangement contrast

---

# 🔧 CHANGE 2 — PRO MODE TONE

Добавить глобальное правило:

=== PRO MODE TONE ===

Use direct, professional, decisive language.

FORBIDDEN:

* maybe
* could
* consider
* try

REQUIRED:

* must
* required
* fix
* necessary

Example:

BAD:
"This could be improved"

GOOD:
"This must be fixed to reach commercial level"

# 🔧 CHANGE 3 — STRONGER ENGINEERING ACTIONS

Allow stronger parameter ranges:

* EQ cuts: -2 to -6 dB
* Loudness: -8 to -9 LUFS (Electronic)
* Compression: more assertive if needed

RULE:
Do NOT stay conservative if problem is obvious.

# 🔧 CHANGE 4 — COMMERCIAL STANDARD TARGET

Добавить:

=== COMMERCIAL TARGET RULE ===

All recommendations must aim for:

* club playback systems
* large speakers
* competitive loudness
* punch and clarity

NOT just “better sound”:
→ must be “competitive with released tracks”

# 🔧 CHANGE 5 — AFTER FIX EXPECTATION

В Full Analysis добавить:

=== AFTER FIX EXPECTATION ===

* louder perceived energy
* tighter low-end
* clearer transients
* improved club translation

Example:

After applying these fixes, the track will hit harder on large systems, with a tighter low-end and more controlled dynamics.
