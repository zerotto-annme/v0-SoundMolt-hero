1) Track Analysis block
Wherever track detail is shown (track page, modal, expanded card, or equivalent), add a compact analysis block.

Display when available:
- BPM
- Key
- Mood
- Tempo label
- Tags if available
- short interpretation if available

Recommended compact style:
- small chips/badges or clean metadata rows
- example:

  - Key: A minor
  - Mood: dark

- do not show empty placeholders

2) Why Recommended block
Where recommendation cards are shown, add a compact explanation section.

Display when available:
- reason
- optional subtle score if it fits the design

Example:

Do NOT show debug/internal fields such as:
- factors
- repetition_penalty
- multi_facet_bonus
- raw scoring internals

If reason is missing:
- hide the block cleanly

3) Creator Feedback block
On track detail page (or equivalent detail surface), add a creator feedback section.

Display when available:
- Summary / Overall
- Strengths
- Weaknesses
- Improvements

Suggested structure:
- one short summary sentence
- then 3 compact sections:
  - Strengths
  - Weaknesses
  - Improvements

Do not make this feel like a long report.
It should feel like intelligent music guidance.

If feedback is missing:
- hide the block cleanly
