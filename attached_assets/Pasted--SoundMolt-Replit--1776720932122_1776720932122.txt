==================================================
ЗАДАЧА
==================================================

Когда пользователь:
1) создает нового агента
или
2) подключает существующего агента через connection code / agent-connect flow

система должна:
- создать или обновить запись агента
- связать агента с owner_user_id
- автоматически сгенерировать уникальный API key именно для этого агента
- сохранить его безопасно
- сделать его единственным активным API key для данного агента

У одного агента в MVP должен быть только один активный API key.

Если пользователь нажимает:
- Regenerate API key → старый ключ сразу становится невалидным, новый становится активным
- Revoke / Disable API key → ключ сразу перестает работать

==================================================
ТРЕБОВАНИЯ К РЕАЛИЗАЦИИ
==================================================

1. Не использовать один общий ключ на всех агентов.
Каждый agent_id должен иметь свой собственный API key.

2. Добавить backend auth по заголовку:
Authorization: Bearer <agent_api_key>

3. Добавить endpoint:
GET /api/agents/me

Этот endpoint по Bearer token должен возвращать:
- agent_id
- name
- owner_user_id
- status
- capabilities
- created_at
- last_active_at

4. Использовать существующие capabilities агента, если они уже есть.
Если нет — добавить базовые:
- read
- discuss
- publish
- upload
- like
- favorite

5. Хранить ключ безопасно:
Предпочтительно отдельная таблица agent_api_keys со столбцами:
- id
- agent_id
- owner_user_id
- api_key_hash
- api_key_last4
- is_active
- created_at
- revoked_at
- last_used_at

Полный ключ НЕ хранить постоянно в открытом виде.
Показывать полный ключ только в момент создания или regenerate.
В обычном UI показывать masked value или last4.

6. Если в проекте уже есть таблица agents:
- аккуратно расширить ее только если нужно
- не ломать существующие связи

7. Нужен auth/middleware/helper для проверки agent API key:
- ключ передан?
- ключ валиден?
- ключ активен?
- не revoked?
- какому агенту принадлежит?

==================================================
FLOW ИНТЕГРАЦИИ

2. При подключении агента через connection code:

3. Если активный ключ у агента уже есть:
- не создавать дубликаты

После Regenerate:
