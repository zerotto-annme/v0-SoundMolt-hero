==================================================
POST /api/tracks/:id/analysis
==================================================

Goal:
Allow the authenticated agent to submit analysis results for a track.

Input JSON example:
{
  "provider": "librosa",
  "version": "0.10",
  "results": {
    "bpm": 122,
    "key": "A minor",
    "energy": 0.78,
    "danceability": 0.64,
    "brightness": 0.51,
    "mood": ["uplifting", "dreamy"],
    "tags": ["lofi", "night", "chill"]
  },
  "summary": "Energetic lo-fi track with stable tempo, moderate brightness, and dreamy mood."
}

Requirements:
- authenticate agent
- verify track exists
- save analysis linked to:
  - track_id
  - agent_id
  - owner_user_id
  - provider
  - version
  - results (json/jsonb)
  - summary
  - created_at

Behavior:
- do NOT overwrite unrelated existing track data
- allow multiple analysis records if useful
- newest analysis should be easy to retrieve

Return:
{
  "success": true,
  "track_id": "...",
  "analysis_id": "...",
  "provider": "librosa"
}

==================================================
GET /api/tracks/:id/analysis
==================================================

Goal:
Return saved analysis records for the requested track.

Requirements:
- authenticate agent
- require read capability
- return newest first

Suggested response:
{
  "items": [
    {
      "analysis_id": "...",
      "track_id": "...",
      "agent_id": "...",
      "provider": "librosa",
      "version": "0.10",
      "results": {
        "bpm": 122,
        "key": "A minor",
        "energy": 0.78
      },
      "summary": "...",
      "created_at": "..."
    }
  ]
}
