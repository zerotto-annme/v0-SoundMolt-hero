Goal:
Turn SoundMolt’s music intelligence into useful feedback for creators.

This is NOT full autonomous mastering.
This is a structured AI feedback layer that interprets track analysis into actionable creator guidance.

==================================================
REQUIRED LOGIC
==================================================

1. Strengths
Use existing analysis/taste signals to highlight what the track does well.

Examples:
- strong BPM alignment
- strong key alignment
- strong mood alignment
- good genre fit
- coherent identity
- recommendation-friendly profile

2. Weaknesses
Use cautious, non-destructive phrasing.
Do not pretend to be a mastering engineer.
Focus on music intelligence interpretation.

Examples:
- too close to recent patterns
- weak novelty
- mood/tempo too narrow for broader discovery
- outside preferred range in some dimensions
- lacks diversity against recent behavior trends

3. Improvements
Return actionable but light-touch suggestions.
These should be framed as guidance, not absolute truth.

Examples:
- vary tempo if broader appeal is desired
- shift mood if diversity is needed
- strengthen contrast if repetition is too high
- consider changing key or energy profile to stand out

4. Fit score
Provide a compact numeric or normalized score showing how well the track fits the current system/taste profile.
Do not overclaim scientific precision.

5. Rebuild endpoint
POST /api/tracks/:id/feedback/rebuild should:
- recompute feedback
- overwrite/update stored feedback snapshot if you store one
- return the rebuilt feedback

6. Storage model
Use the simplest clean implementation:
- either compute on read
- or store a feedback snapshot table

If storage is added, keep it minimal, for example:
- track_feedback(id, track_id, owner_user_id, provider, payload jsonb, created_at, updated_at)

Do not overengineer.
