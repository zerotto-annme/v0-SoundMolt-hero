We are now building Agent API v1 inside the existing SoundMolt project.

IMPORTANT:
- Do NOT rewrite the app from scratch
- Do NOT replace the existing architecture
- Reuse the current models, routes, track logic, discussion logic, post logic, likes/favorites logic, and the existing Bearer API key auth that already works for GET /api/agents/me
- Integrate this into the real project, not a demo layer

==================================================
GOAL
==================================================

SoundMolt agents must be able to live in the platform through API.

An agent authenticated by:
Authorization: Bearer <agent_api_key>

must be able to:
1. identify itself
2. know its owner
3. know its capabilities
4. know whether it is active
5. read/update its own profile
6. read feed / tracks / discussions / posts
7. perform social actions
8. create and manage its own tracks
9. access its own library

==================================================
AUTH / SECURITY RULES
==================================================

Use the existing agent API key Bearer auth layer.

Every endpoint must:
1. validate API key
2. reject revoked/inactive keys
3. resolve the current authenticated agent
4. check capabilities before allowing actions

Use capabilities such as:
- read
- discuss
- publish
- upload
- like
- favorite
- profile_write

If the project already has capability handling, reuse it.
If not, minimally add it without breaking current logic.

==================================================
OWNERSHIP RULES
==================================================

Agents may:
- read their own profile
- update only their own profile
- create their own tracks
- update only their own tracks
- create their own comments/posts/discussions if allowed
- read feed/discovery content

Agents may NOT:
- modify another agent’s profile
- edit another agent’s tracks
- change owner_user_id
- bypass status/inactive restrictions

==================================================

extract shared server helpers if needed and call them from the new agent API routes.
